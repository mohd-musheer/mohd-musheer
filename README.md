## Featured projects

### [VectorForgeML](https://github.com/mohd-musheer/VectorForgeML)

High-performance R and C++ machine-learning framework built from scratch around modular, systems-level ML.

- Implements core supervised and unsupervised algorithms without heavy ML dependencies.
- Uses BLAS/LAPACK for linear algebra and OpenMP for multi-core execution.
- Provides R bindings with a C++ backend and modular preprocessing/pipeline components.
- Published as an R package on [CRAN](https://cran.r-project.org/package=VectorForgeML).

### [CodeShield-AI](https://github.com/mohd-musheer/CodeShield-AI)

Repository security analyzer that combines static checks and LLM-assisted code understanding to surface risks.

- Scans public GitHub/GitLab repositories, local directories, and ZIP uploads.
- Detects vulnerabilities, exposed secrets, insecure configuration, and dependency-related risks.
- Produces explainable findings, suggested fixes, and downloadable reports in formats including SARIF, JSON, CSV, Markdown, and HTML.
- Built around FastAPI, Groq-powered analysis, Docker, and CI/security workflows.

### [AI Resume Ranking System](https://github.com/mohd-musheer/AI-Resume-Rranking-System)

NLP-based, ATS-style candidate screening tool that ranks resumes against a role and its required skills.

- Uses TF-IDF and cosine similarity to compare resumes with a job description.
- Identifies matched and missing skills alongside ranking results for clearer review.
- Supports CSV resume uploads and role-specific skill inputs through a web interface and REST API.
- Containerized with Docker and implemented with Python, FastAPI, and scikit-learn.

### [Axion](https://github.com/mohd-musheer/Axion)

Performance-oriented LLM inference and serving runtime exploring efficient execution from the systems layer up.

- Includes C++ runtime components for memory management, execution, streaming, and KV-cache handling.
- Explores quantization, batching, and CPU/GPU-oriented inference paths.
- Includes benchmarking work for latency, throughput, memory, and speculative decoding.
- Organizes deployment and inference configuration for scalable runtime experimentation.

### [MyContextAI](https://github.com/mohd-musheer/MyContextAI)

Private personal RAG assistant that builds a searchable AI context from publicly supplied profiles and documents.

- Ingests GitHub repositories, portfolio sites, resumes, blogs, and custom URLs.
- Chunks and embeds collected content into persistent ChromaDB storage.
- Uses OpenRouter-backed LLM responses through a FastAPI API and web interface.
- Ships with Docker support while keeping the indexed source set under the user’s control.

### [OpenDocs-AI](https://github.com/mohd-musheer/OpenDocs-AI)

Documentation copilot that answers questions over technical docs using a vectorless RAG approach.

- Crawls configured documentation sources and converts them into structured Markdown.
- Builds PageIndex document trees instead of relying on embeddings or a vector database.
- Retrieves relevant documentation context for Groq-powered answers through a web UI and REST API.
- Supports multiple documentation sources, memory components, and Dockerized deployment.
